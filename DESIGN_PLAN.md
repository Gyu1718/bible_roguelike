# 이름 없는 증인들 UI 리디자인 설계안

## 목표

현재 게임은 기능 프로토타입 단계이다. 다음 단계는 이미지 콘셉트 3개를 기준으로 화면 역할을 분리하고, 실제 웹게임 UI 구조로 정리하는 것이다.

적용 방향은 다음과 같다.

```text
메인 / 챕터 선택 화면 → 벽돌공과 밤의 도시가 있는 다크 픽셀-페인터리 메뉴
플레이 화면 → 홍해 앞 선택지 화면
엔딩 화면 → 해방의 증인 결과 화면
```

---

# 1. 전체 페이지 구조

## 1) Home / Chapter Select

역할:

- 게임 첫인상 제공
- 챕터 선택
- 기록 / 증인 / 유물 / 업적 / 설정 진입점
- 현재 진행 중인 챕터 표시

적용 이미지 콘셉트:

- 벽돌공 장면이 있는 다크한 챕터 메뉴 화면

핵심 분위기:

```text
억압
밤
벽돌 노동
제국의 도시
다크 픽셀 일러스트
인디 로그라이크 메뉴
```

화면 구성:

```text
상단:
- 게임 타이틀: 이름 없는 증인들
- 부제: 기억하라, 기록하라, 전하라

좌측:
- 세로 메뉴
  - 챕터
  - 증인
  - 기록
  - 유물
  - 업적
  - 설정

중앙/우측:
- 챕터 카드 목록
  - 1장 벽돌과 바다
  - 2장 광야의 메아리
  - 3장 무너진 성벽
  - 4장 불 가운데서
  - 5장 기록된 이름

하단:
- 짧은 인용문 또는 세계관 문장
```

현재 구현 우선순위:

```text
1순위: 챕터 선택 카드
2순위: 시작하기 버튼
3순위: 기록/업적 메뉴는 비활성 상태로 표시
```

# 2. Main Gameplay Screen

역할:

- 실제 게임 플레이
- 장면 삽화
- 서사 텍스트
- 선택지
- 주사위 판정 선택 표시
- 상태 / 기록 로그 표시

적용 이미지 콘셉트:

- 홍해 앞 선택지 화면

핵심 분위기:

```text
거대한 사건 앞의 작은 인간
어두운 바다
횃불
공포와 기다림
선택의 긴장
```

화면 구성:

```text
좌측:
- 큰 장면 일러스트
- 장면 분위기를 전달하는 영역

중앙:
- 챕터명
- 장면명
- 서사 본문
- 선택지 3~4개
- 판정 선택에는 작은 주사위 아이콘만 표시

우측:
- 상태
  - 생존
  - 공포
  - 증언
- 기록 로그

하단:
- 보조 버튼
  - 새 회차
  - 처음으로
```

선택지 UX 규칙:

```text
일반 선택:
- 선택 즉시 다음 장면으로 진행
- 결과는 오른쪽 기록에만 남김

주사위 선택:
- 선택지 오른쪽에 주사위 아이콘 표시
- 선택 시 2d6 판정 화면으로 이동

치명 선택:
- 별도 위험 표시 없음
- 선택 후 결과창에서 즉시 배드엔딩으로 드러남
```

# 3. Roll Check Screen

역할:

- 주사위 판정 전용 화면
- 위험도 높은 선택의 긴장감 부여

화면 구성:

```text
좌측:
- 현재 장면 이미지 유지

중앙:
- 선택한 행동 표시
- 2d6 결과 표시
- 보정치 표시
- 목표값 표시
- 성공 / 실패 표시

우측:
- 상태 유지
- 기록 로그 유지
```

현재 규칙:

```text
위험도 70 이상 100 미만 → 2d6 판정
위험도 100 → 즉시 배드엔딩
```

---

# 4. Feedback Screen

역할:

- 모든 선택에 표시하지 않는다.
- 중요한 선택, 판정 결과, 치명 선택, 클라이맥스 선택에만 표시한다.

화면 구성:

```text
상단:
- 선택의 결과

중앙:
- 결과 제목
- 결과 설명

하단:
- 계속
- 배드엔딩 확인
```

사용 기준:

```text
일반 선택 → 표시하지 않음
증언이 크게 오르는 선택 → 표시 가능
공포가 크게 오르는 선택 → 표시 가능
주사위 판정 이후 → 표시
치명 선택 → 표시 후 배드엔딩
```

---

# 5. Ending Screen

역할:

- 감정적 보상
- 엔딩 결과 제시
- 숨겨진 증언 유형 표시
- 재도전 유도

적용 이미지 콘셉트:

- 해방의 증인 엔딩 화면

핵심 분위기:

```text
해방
빛
물벽
상징적 구원
장엄한 마무리
```

화면 구성:

```text
배경:
- 홍해 도하 이후의 밝은 길
- 물벽과 빛
- 상징적 문양

중앙 패널:
- 엔딩명
- 엔딩 본문
- 당신의 증언 유형
- 증언 유형 해석문

버튼:
- 다시 도전
- 처음으로
```

현재 증언 유형:

```text
두려움 속의 순종자
해방의 증언자
상처 입은 생존자
공포에 흔들린 사람
무모한 저항자
피하려는 생존자
```

# 6. 파일 구조 제안

현재 안정판 구조:

```text
index.html
styles.css
game-data.js
game.js
```

다음 단계 구조:

```text
index.html
styles.css
game-data.js
game.js
ui.js
profile.js
roll.js
```

## 파일별 역할

### index.html

- 페이지 뼈대
- CSS와 JS 연결

### styles.css

- 전체 비주얼 스타일
- 홈 / 플레이 / 엔딩 화면 레이아웃
- 버튼, 카드, 패널, 아이콘 스타일

### game-data.js

- 챕터 데이터
- 장면 데이터
- 선택지 데이터
- 엔딩 데이터
- 숨은 증언 유형 점수 데이터

### game.js

- 전체 게임 상태 관리
- 화면 전환
- 선택 처리
- 엔딩 판정

### ui.js

- 화면 렌더링 전담
- homeScreen()
- playScreen()
- rollScreen()
- feedbackScreen()
- endingScreen()

### profile.js

- 숨은 증언 유형 계산
- 엔딩에서 증언 유형 출력

### roll.js

- 2d6 판정
- 목표값 계산
- 보정치 계산

# 7. 단계별 리디자인 순서

## 1단계

현재 코드 유지하면서 styles.css만 개선한다.

목표:

```text
메뉴 화면 분위기 강화
선택지 가독성 개선
주사위 아이콘 정돈
오른쪽 상태창 정리
```

## 2단계

Home / Play / Ending 화면에 서로 다른 레이아웃 클래스를 부여한다.

```text
home-screen
play-screen
ending-screen
```

## 3단계

game.js에서 UI 렌더링 함수를 ui.js로 분리한다.

## 4단계

엔딩 화면을 별도 스타일로 강화한다.

```text
밝은 물벽 배경
중앙 결과 패널
증언 유형 카드
```

## 5단계

챕터 선택 화면을 본격적으로 만든다.

```text
챕터 카드
잠김 상태
진행 상태
기록 보관소 버튼
```

# 8. 최종 방향 요약

이 게임의 비주얼 방향은 하나로 통일하기보다, 화면 역할에 따라 달라져야 한다.

```text
메인 메뉴:
어둡고 억압적인 벽돌 노동 세계

플레이 화면:
가독성 높은 다크 시네마틱 선택지 UI

엔딩 화면:
밝고 장엄한 해방의 상징 UI
```

이렇게 나누면 게임의 감정선이 분명해진다.

```text
억압에서 시작한다.
선택 속에서 흔들린다.
해방의 증언으로 끝난다.
```
